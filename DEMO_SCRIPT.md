## Demo Script (5-7 minutes)

### Segment 1: App Introduction (0:00 - 0:30)
```
NARRATION:
"Welcome to Rice Quality Analyzer - an AI-powered mobile application 
for field-level rice quality assessment, developed for the UNIDO 
AfricaRice Challenge."

ACTIONS:
1. Show app icon on home screen
2. Tap to launch app
3. Brief pause on welcome screen
```

### Segment 2: User Signup (0:30 - 1:30)
```
NARRATION:
"The app starts with a user-friendly onboarding flow that includes
an important disclaimer about the application's intended use."

ACTIONS:
1. Tap "Get Started"
2. Read disclaimer briefly (pause on key points)
3. Accept disclaimer
4. Fill profile form:
   - Name: "Demo User"
   - Role: Select "Quality Assessor"
   - Organization: "Rice Mill Co."
5. Submit and navigate to home
```

### Segment 3: Home Screen Overview (1:30 - 2:00)
```
NARRATION:
"The home screen provides quick access to start a new scan and 
shows recent activity. Notice the clean, simple interface designed 
for field workers."

ACTIONS:
1. Show home screen elements
2. Point out "New Scan" button
3. Show quick stats section
4. Show empty recent scans (first time use)
```

### Segment 4: Camera Capture (2:00 - 3:30)
```
NARRATION:
"Now let's capture a rice sample. The app provides guidance to help
users take quality photos for accurate analysis."

ACTIONS:
1. Tap "New Scan"
2. Grant camera permission
3. Show camera interface with:
   - Rice type selector (select "White Rice")
   - Guidance overlay text
4. Position rice sample on blue background
5. Capture photo
6. Show preview screen with validation results
```

### Segment 5: Analysis & Results (3:30 - 4:30)
```
NARRATION:
"Watch as our AI model analyzes the rice sample. This runs entirely
on-device, providing results in seconds without needing internet 
connectivity."

ACTIONS:
1. Tap "Analyze" button
2. Show loading animation (wait ~2-3 seconds)
3. Results appear - show:
   - Grade indicator (e.g., "Grade 1")
   - Summary statistics
   - Any warnings
4. Scroll to show all metrics
5. Tap "View Details" for full breakdown
```

### Segment 6: Detailed Results (4:30 - 5:00)
```
NARRATION:
"The detailed view provides comprehensive metrics including grain
counts, color composition, kernel dimensions, and CIELAB color profile -
all the data needed for quality assessment."

ACTIONS:
1. Scroll through detailed metrics
2. Highlight key sections:
   - Grain distribution chart
   - Color defects breakdown
   - Kernel measurements
```

### Segment 7: History & Export (5:00 - 5:45)
```
NARRATION:
"All scans are saved locally on the device. Users can view history,
view past results, and export data to CSV for record-keeping."

ACTIONS:
1. Navigate to History tab
2. Show the scan we just created
3. Tap on a scan to view again
4. Navigate back
5. Demonstrate export:
   - Tap export button
   - Show share sheet with CSV option
```

### Segment 8: Offline Mode Demonstration (5:45 - 6:30)
```
NARRATION:
"A critical feature for field use in rural Ghana - the app works
completely offline. Let me demonstrate by enabling airplane mode."

ACTIONS:
1. Pull down notification shade
2. Enable Airplane Mode
3. Show offline indicator in app
4. Navigate to capture
5. Take another photo
6. Run analysis (still works!)
7. Show results (works offline)
8. Disable airplane mode
```

### Segment 9: App Info & Closing (6:30 - 7:00)
```
NARRATION:
"The app includes proper attribution and credits. Thank you for
watching our demonstration of Rice Quality Analyzer - bringing
AI-powered rice quality assessment to the field."

ACTIONS:
1. Navigate to Info tab
2. Show app version and model version
3. Show UNIDO and AfricaRice credits
4. Show disclaimer text
5. Return to home screen
6. End recording
```

- [ ] Add intro/outro cards if needed

- [ ] Export at 1080p minimum
- [ ] Final runtime check (target: 5-7 minutes)

## Fallback Plans

### If rice sample not available:
- Use the app's mock inference (always available)
- Mention it's using sample data for demonstration

### If camera issues:
- Use gallery picker to load test image
- Pre-capture images as backup

### If app crashes:
- Have APK ready to reinstall
- Keep spare device prepared
